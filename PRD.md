# 多校区教育培训机构信息管理系统 PRD

**版本**：V1.0  
**日期**：2026-03-05  

## 1. 背景与问题
教育培训机构在多校区扩张、课程体系多样化、学员规模增长的背景下，仍普遍采用“Excel+群消息+纸质登记”的组合方式进行管理，导致数据分散、重复录入、流程割裂与追溯困难。典型表现包括：学员档案不统一、报名缴费与教学执行脱节、课消与余额难以对齐、统计分析依赖人工汇总等。
本产品旨在通过信息化手段打通“学员—课程—报名缴费—教学执行—统计分析—在线学习”的业务闭环，实现多角色协同、多校区数据隔离与关键操作可追溯，提升运营效率与管理透明度。

## 2. 产品目标与成功指标

### 2.1 产品目标
- 建立统一学员主数据：建档、绑定关系、状态变更可追溯。
- 打通报名缴费与教学执行：订单、考勤、课消与课时余额一致。
- 支持多校区运营：总部全局视图与校区隔离并存。
- 提供统计分析能力：核心经营/教学指标可筛选、可导出。
- 降低操作成本：导入导出任务化、权限边界清晰。

### 2.2 成功指标
| 指标 | 口径说明 | 目标 |
| :--- | :--- | :--- |
| 核心流程完成率 | 报名→订单→收款→排课→考勤→课消 | ≥95% 用例通过 |
| 列表页响应时间 | 学员/订单等分页列表 | 95% 请求 ≤2.5s |
| 权限越权缺陷 | 跨校区/非授课/非绑定访问 | 0 个 P0 |
| 课消一致性 | 课消记录与订单余额对齐 | 一致性校验通过 |
| 导入导出可用性 | 任务创建/状态/下载 | 成功率 ≥98% |

## 3. 用户与角色（Persona）
系统面向四类核心角色：总部管理员、校区管理员、教师、学员/家长。

| 角色 | 核心诉求 | 主要使用场景 |
| :--- | :--- | :--- |
| 总部管理员 | 全局配置与跨校区经营分析 | 校区/角色权限管理、全局报表、审计查看 |
| 校区管理员 | 本校区日常运营与教务执行 | 建档、报名收款、排课、课消、校区报表、导入导出 |
| 教师 | 教学执行与课堂管理 | 查看课表、考勤登记、资源发布、查看班级学员 |
| 学员/家长 | 课程安排与学习过程透明 | 查看订单/课时、课表、考勤/课消、在线学习与测验 |

## 4. 使用范围与版本规划

### 4.1 V1 范围（最小可用闭环）
- 学员管理（建档、绑定、状态、附件）
- 课程与班级（课程、班级、教师分配、容量）
- 报名缴费（报名、订单、收款、退费、课时余额）
- 教学管理（课表/课次、考勤、课消）
- 统计报表（招生/营收/到课率/课消率，支持导出）
- 在线学习（简化版：章节/资源/进度/测验）
- 通用支撑：JWT登录、RBAC、数据范围过滤、审计日志、导入导出异步任务

### 4.2 V2 可扩展方向
- 更完善的指标体系与离线汇总
- 转班/插班/补课、套餐与优惠券、合同与发票流程细化
- 移动端能力增强（推送提醒、上课签到、家校沟通）

## 5. 核心业务流程
- **核心闭环 (P0)**：线索/咨询 → 学员建档 → 课程/班级选择 → 报名确认 → 生成订单 → 缴费收款 → 排课/课表 → 上课执行 → 考勤登记 → 课消确认（扣课时） → 统计报表 → 续费/转介绍
- **异常流程 (P0)**：退费/转班/停课：订单状态更新 → 课时余额调整 → 审计日志记录 → 报表口径同步
