## 🔴 严重问题 (Critical)

### 1. 学习进度数据严重不足

**现象**:
```
8 条学习进度记录
vs
1080 条考勤记录
vs
155 个订单
vs
73 个学员

覆盖率: 仅 1.4% (1/73 学员有进度)
```

**问题**:
- StudentLessonProgress 表未被业务流程正确写入
- 缺少 attendance -> lesson_progress 映射逻辑
- 学员无法查看自己的学习进度

**修复方案**: 
在考勤扣费成功后，自动创建学习进度记录，预计可补充 ~900 条记录。

详见: LINKAGE_AUDIT_REPORT.md 第六章 优先级1

## 🏷️ 技术细节

**数据库**: SQLite (`prisma/dev.db`)  
**ORM**: Prisma v6.19.2  
**Runtime**: Node.js v22+  
**数据模型**: 详见 `prisma/schema.prisma`

**关键表**:
- `SysUser`: 用户
- `EduStudent`: 学员档案
- `EdClass`: 班级
- `EdCourse`: 课程
- `FinOrder`: 订单
- `FinAssetAccount`: 课时资产
- `StudentLessonProgress`: **学习进度** (关键)
- `TeachAttendance`: 考勤记录
- `StdCourseStandard`: 课程标准
- `StdCourseChapter`: 章节
- `StdCourseLesson`: 课时
